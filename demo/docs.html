Quickstart

Drift is a single-binary version-control system. From install to your first synced commit in three commands.

Install

One binary, no dependencies. Pick your platform:

# macOS / Linux
curl -fsSL https://drift.example/install.sh | sh

# Windows (PowerShell)
irm https://drift.example/install.ps1 | iex

The installer drops a drift binary in ~/.local/bin (Unix) or %LOCALAPPDATA%\Drift (Windows). Confirm:

$ drift --version
drift 1.0.0 (2026-05-31, deadbeef)

Init a repo

In any directory:

$ cd ~/projects/my-project
$ drift init
Initialized drift repository in .drift/

Drift's metadata lives in .drift/. Add it to your gitignore if you're co-existing with Git for now; the two don't share state.

Drift and Git side-by-side You can run both VCSes against the same working tree without conflicts. Drift writes only to .drift/; Git writes only to .git/. The team's been doing this in production for six months without an incident.

Make your first commit

Track everything, then commit:

$ drift track .
3,841 files tracked.

$ drift commit -m "Initial commit"
Committed: f3a91b2 (3,841 files, 4.2 MB)

Sync to a remote

Drift remotes are URLs over HTTPS. Set one + push:

$ drift remote add origin https://drift.example/me/my-project
$ drift sync
Pushing to origin... 4.2 MB transferred in 1.8s.

The remote stores your commits in a content-addressable blob store; nothing about the directory layout, your filenames, or your branch structure is exposed in the URL.

Branch + merge

Branches in Drift are first-class names with no inherent hierarchy. Create one:

$ drift branch new feature/login
Switched to branch 'feature/login'

Make changes, commit, switch back, merge:

$ drift commit -m "Add login form"
Committed: 2e1f8c4

$ drift branch switch main
Switched to branch 'main'

$ drift merge feature/login
Merged 1 commit. No conflicts.

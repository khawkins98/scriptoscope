Chrome rendered from real extracted bitmaps via @aaron-ui/scheme-extractor. Each theme uses a different CSS strategy informed by what its native chrome composite actually contains. 7 Le's 74×25 widget-strip cicn is positioned via pseudo-elements that clip left and right widget kits to the titlebar's edges, pinstripe synthesized in CSS for the middle. ErgoBox's 132×64 full-frame cicn renders at native pixel size (3× scale) so the projecting BeOS-tab structure survives; the trade-off is fixed-size windows — variable-size ErgoBox windows would need a richer bundle format that separates tab geometry from body frame. You'll notice ErgoBox bodies render white, not the medium gray of the masswerk reference thumbnail: this is faithful to the cicn — Kaleidoscope composes a separate ppat over the body at runtime to produce the gray, which our demo doesn't yet reproduce. Welcome

This window's chrome is rendered from the actual scheme bitmaps — no synthetic gradients, no SVG primitives. The close box, zoom box, and windowshade arrow you see at the corners come straight from the author's hand-pixeled artwork.

The two themes use different CSS strategies because their chrome cicns ship in different shapes. 7 Le's is a 74×25 widget strip; ErgoBox's is a 132×64 full-frame composite.

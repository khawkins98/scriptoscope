Scheme Diagnostics

Per-scheme view of how Aaron UI decodes + renders a Kaleidoscope scheme. Shows the raw chrome cicns with parts-rect + recipe-position overlays, the classifier verdict, the derived slice geometry, and a live render with optional segment-outline highlighting. Use this to diagnose extraction-vs-rendering gaps: if the recipe data looks correct here but the live render is wrong, the bug is in the renderer. If the data itself is missing or wrong, the bug is in the extractor.

Classifier verdict

Chrome cicns

Each chrome state cicn at 4× scale. Toggle overlays to see how the renderer interprets the bitmap.

Recipes (wnd#)

For each side: list of {at, part} entries. at is the cicn-pixel position along the side axis. part matching a slug in the parts table = named (renders that rect at native size at at). part not in the table = fill (tiles cicn pixels in the segment).

Parts (named cicn rects)

Slugs referenced by recipe entries. Each rect [left, top, right, bottom] in cicn pixels.

Live render

A real AaronWindow with this scheme applied. Toggle "Highlight segments" to outline each recipe-rendered piece in the rendered chrome (named parts in pink; fill segments in blue).

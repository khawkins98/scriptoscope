Eighteen schemes. One runtime.

Scriptoscope plays Kaleidoscope schemes — the 1997–2003 Mac OS theming format — the way an emulator plays a ROM: pixel-faithfully, in your browser, from the original binary resources their authors shipped. Drop a script tag on any page, add a few attributes, and ship 1997 to production.

<!-- one tag, eighteen scheme bundles -->
<script type="module" src="scriptoscope.js"></script>
<article data-scriptoscope-window data-scriptoscope-title="Hello">…</article>

kDEF Replay Engine

A clean-room reimplementation of the 2.3.1 WDEF Kaleidoscope shipped — same part-code jump table, same wnd# rect-list walk, same nine-slice of every cicn. Every pixel of chrome is composited from the binary's own resources.

✎

data-scriptoscope-*

The attribute set that lets any element declare what it is — a window, a button, a checkbox, a slider — and wear the active scheme's chrome the instant the runtime mounts.

<button data-scriptoscope-button>OK</button>
<input type="range" data-scriptoscope-slider />
<article data-scriptoscope-window>…</article>
↘

ResourceForkLib

Reads any classic scheme — a StuffIt archive, a MacBinary wrapper, a raw .rsrc fork still warm from a System 7 disk — directly in your browser. Nothing uploaded.

✦

The Scheme Library

Eighteen of the era's best — 1138, Apple Platinum, BeOS R5, Crayon OS, Monkey Paradise, Windows 95, and twelve more. Decoded from archives the Macintosh Garden preserves.

Reverse-engineered clean-room from the decompiled Kaleidoscope 2.3.1 kDEF — no original code is shipped or executed. Deliberate divergences are catalogued in the faithfulness ledger.
